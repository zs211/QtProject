# MyFirstQtApp - 智能历史记录管理工具

## 项目简介
这是一个基于 Qt6 开发的桌面应用，集成了**历史记录管理、数据导出、多窗口交互、网络内容获取**等功能，适用于日常数据记录与管理场景。

## 功能清单
- ✅ **历史记录管理**：增删改查、关键词搜索、一键清空
- ✅ **数据导出**：支持导出为 TXT/CSV 格式，方便数据备份与分析
- ✅ **多窗口交互**：可打开子窗口并传递数据
- ✅ **网络功能**：获取网页内容（支持 HTTPS）
- ✅ **独立运行**：已打包所有依赖，无需安装 Qt 环境

## 运行说明
### 环境要求
- Linux (Ubuntu 20.04+/Debian 10+)
- 无需安装 Qt 或其他依赖库

### 快速启动
1. 解压 `package.zip` 到任意目录
2. 进入 `package` 文件夹
3. 执行启动脚本：
   ```bash
   ./run.sh
   ```
   或直接双击 `run.sh` 文件

### 目录结构
```
package/
├── MyFirstQtApp          # 主程序可执行文件
├── run.sh                # 启动脚本（配置依赖环境变量）
├── libs/                 # Qt 核心库 + OpenSSL 依赖库
├── platforms/            # Linux 窗口平台插件（libqxcb.so）
├── sqldrivers/           # SQLite 数据库驱动插件
└── tls/                  # TLS/SSL 后端插件（支持 HTTPS）
```

## 使用指南
1. **添加记录**：在「请输入」框中输入内容，点击「确认」保存到历史记录
2. **搜索记录**：在搜索框输入关键词，点击「搜索」筛选历史记录
3. **导出数据**：点击「导出TXT」或「导出csv」，选择保存路径即可导出
4. **清空记录**：点击「清空历史记录」，二次确认后清空所有数据
5. **打开子窗口**：点击「打开子窗口」，可传递当前输入内容到新窗口
6. **获取网页**：点击「获取网页内容」，将自动获取百度首页源码并显示

## 技术栈
- **开发框架**：Qt 6.11.0 (C++)
- **数据库**：SQLite3（轻量级本地存储）
- **网络模块**：Qt Network（支持 HTTPS/TLS）
- **打包方式**：手动拷贝依赖 + 启动脚本（兼容主流 Linux 发行版）

## 常见问题
### 1. 运行时提示 "缺少库文件"
请确保 `libs/`、`platforms/`、`sqldrivers/`、`tls/` 目录与 `MyFirstQtApp`、`run.sh` 在同一级目录，不要修改目录结构。

### 2. 网络功能提示 "TLS 初始化失败"
已在打包时集成 OpenSSL 依赖，正常运行 `run.sh` 即可解决，无需额外安装。

### 3. 界面警告（不影响功能）
终端中可能出现以下无害警告，可直接忽略：
- `No matching signal for on_action_Exit_triggered()`：信号槽自动匹配警告
- `dbus reply error`：Linux 桌面主题相关警告，不影响程序逻辑
